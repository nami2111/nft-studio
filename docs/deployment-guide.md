# Deployment Guide

GNStudio is deployed on the Juno platform, a serverless infrastructure built on the Internet Computer Protocol (ICP).

## Architecture

- **Hosting**: Static site hosted on a Juno satellite
- **Analytics**: Juno Orbiter for usage analytics
- **Build**: Vite+ static adapter, output to `build/` directory
- **Domain**: Served via Juno's decentralized CDN

## Build & Deploy

### Pre-deploy Build

The configuration in `juno.config.ts` specifies:

```typescript
{
	satellite: {
		source: 'build',          // Build output directory
		predeploy: ['vp run build'] // Command run before deploy
	}
}
```

### Deployment Commands

```bash
# Build for production
vp run build

# Deploy to production
juno hosting deploy

# Deploy with configuration applied
juno hosting deploy --config

# Clear existing files before deploy
juno hosting deploy --clear

# Deploy with progress streaming
juno hosting deploy --progress
```

### Full Deploy Workflow

```bash
# 1. Run all checks
vp check

# 2. Build for production
vp run build

# 3. Deploy to Juno
juno hosting deploy
```

## CSP Headers

Content Security Policy headers are configured in `juno.config.ts` and applied to all served files:

```
default-src 'self'
script-src 'self' 'unsafe-inline'
style-src 'self' 'unsafe-inline'
img-src 'self' data: blob: https:
connect-src 'self' https: wss:
font-src 'self' data: https:
object-src 'none'
base-uri 'self'
frame-ancestors 'none'
worker-src 'self' blob:
```

### Key Points

- **Worker Support**: `worker-src 'self' blob:` allows Web Workers to be created from blob URLs (required for the worker pool)
- **Image Support**: `img-src data: blob:` allows image data URLs and blob URLs for trait previews
- **Font Loading**: `font-src data: https:` allows both self-hosted and Google Fonts
- **No Framing**: `frame-ancestors 'none'` prevents clickjacking

## Permissions Policy

Restricts browser feature access for security:

```
accelerometer=(), camera=(), geolocation=(), gyroscope=(), magnetometer=(), microphone=(), payment=(), usb=()
```

All hardware APIs are disabled since GNStudio doesn't use them.

## Storage Headers

Content-Type and encoding headers are configured per file type:

| Pattern     | Headers                                                          |
| ----------- | ---------------------------------------------------------------- |
| `**/*.js`   | `Content-Type: application/javascript`, `Content-Encoding: gzip` |
| `**/*.mjs`  | `Content-Type: application/javascript`, `Content-Encoding: gzip` |
| `**/*.wasm` | `Content-Type: application/wasm`                                 |

## Configuration File

The deployment configuration lives at `juno.config.ts`:

```typescript
import { defineConfig } from '@junobuild/config';

export default defineConfig({
	satellite: {
		ids: {
			development: '<DEV_SATELLITE_ID>',
			production: 'dpl4s-kqaaa-aaaal-asg3a-cai'
		},
		source: 'build',
		predeploy: ['vp run build'],
		storage: {
			headers: [
				// JS/MJS headers
				{
					source: '**/*.js',
					headers: [
						['Content-Type', 'application/javascript'],
						['Content-Encoding', 'gzip']
					]
				},
				{
					source: '**/*.mjs',
					headers: [
						['Content-Type', 'application/javascript'],
						['Content-Encoding', 'gzip']
					]
				},
				{ source: '**/*.wasm', headers: [['Content-Type', 'application/wasm']] },
				// CSP and Permissions-Policy for all files
				{
					source: '**/*',
					headers: [
						['Content-Security-Policy', '...'],
						['Permissions-Policy', 'accelerometer=(), ...']
					]
				}
			]
		}
	},
	orbiter: {
		id: 'p2pi7-hiaaa-aaaal-asaia-cai'
	}
});
```

## Environment Configuration

For local development, create a `.env` file:

```bash
# Feature flags for development
VITE_ENABLE_LAYER_REF=true
VITE_ENABLE_ZIP_WORKER_OFFLOADING=true

# Disable streaming for debugging
# VITE_DISABLE_STREAMING_STORAGE=true

# Disable adaptive batch size for testing
# VITE_DISABLE_ADAPTIVE_BATCH_SIZE=true
```

## PWA Deployment

The PWA manifest is generated by `vite-plugin-pwa` from `vite.config.ts`. The manifest references these icon paths:

- `pwa-64x64.png` - Small icon
- `pwa-192x192.png` - Home screen icon
- `pwa-512x512.png` - Splash screen icon
- `maskable-icon-512x512.png` - Adaptive icon with safe zone

Ensure these files exist under `static/` before shipping a production PWA install experience.

The service worker handles:

- Auto-update registration on new version detection
- Google Fonts caching (365 days)
- Asset caching with navigate fallback for offline support
